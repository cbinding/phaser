Introduction

The PHASER application was developed as part of the MATRIX project. It is intended to aid in the editing and analysis of archaeological stratigraphic data. The application requires a modern web browser to function correctly.

[Back to top]

Menu

File - New

This will clear any current data from the diagram and tabs. A pop-up confirmation warning will be displayed before any data is cleared.

[Back to top]

File - Open (JSON)

Open an existing external JSON file containing stratigraphic matrix data compatible with the PHASER application. The JSON file is the main way to store the stratigraphic data, diagram layout and metadata, for data interchange and backup purposes.

[Back to top]

File - Save (JSON)

Save all current matrix data to an external JSON file. NOTE: as this application runs within the browser it is governed by the browser settings. The default settings for saving to external files may be to save directly to a downloads folder with no intervening prompt. If this is the case you can change your browser settings to prompt where to save. On Chrome (as an example) this is under "settings - advanced - downloads". There is an option "ask where to save each file before downloading". Other browsers should have similar facilities.

[Back to top]

File - Import Perio.do collection

Import a collection of named periods from the Perio.do dataset. The entire dataset is downloaded from the Perio.do site and then the application allows you to choose a specific collection to import into your dataset. It is possible to import more than one collection if required. Once imported the periods can be referenced by other elements within the application. Imported periods are included in the JSON file when your matrix data is saved, so do not need to be re-imported.

File - Import CSV

Import data from a delimited text file. Delimited files are UTF-8 text files. Each row of the file represents a data record ending with a newline character, and a specific text character delimits the boundary between fields. The application supports several commonly used delimiter characters (tab, comma, semicolon, pipe, or space). A header row contains delimited field names. These required field names are case-sensitive (siteCode â‰  SiteCode), but the ordering of the fields does not affect the import. Data field values are all case-sensitive. The following tables detail the required fields for each delimited file.

Delimited data import/export format - Phase records
Field name Description
siteCode Identifier of the site this phase falls within
phaseID Unique (within the site) identifier for this phase
minYear Minimum year of the (entered) date range associated with this phase
maxYear Maximum year of the (entered) date range associated with this phase
description Textual description of the phase
withinPeriod Identifier of the period associated with this phase
Delimited data import/export format - Group records
Field name Description
siteCode Identifier of the site this group falls within
groupID Unique (within the site) identifier for this group
groupType Type of group e.g. "hearth", "well", "trench" etc.
cud Construction/use/disuse indicator. Possible values: "", "C", "U", "D", "CU", "UD", "CD", "CUD"
description Textual description of the group
withinPeriod Identifier of the period associated with this group
withinPhase Identifier of the phase this group falls within
Delimited data import/export format - Sub-Group records
Field name Description
siteCode Identifier of the site this subgroup falls within
subgroupID Unique (within the site) identifier for this subgroup
subgroupType Type of subgroup e.g. "hearth", "well", "trench" etc.
cud Construction/use/disuse indicator. Possible values: "", "C", "U", "D", "CU", "UD", "CD", "CUD"
description Textual description of the subgroup
withinPeriod Identifier of the period associated with this subgroupsubgroup
withinGroup Identifier of the group this subgroup falls within
Delimited data import/export format - Context records
Field name Description
siteCode Identifier of the site this context falls within
contextID Unique (within the site) identifier for this context
contextType Type of context e.g. "layer", "cut", "fill" etc.
cud Construction/use/disuse indicator. Possible values: "", "C", "U", "D", "CU", "UD", "CD", "CUD"
description Textual description of the context
withinPeriod Identifier of the period associated with this context
withinPhase Identifier of the phase this context falls within
withinGroup Identifier of the group this context falls within
withinSubgroup Identifier of the subgroup this context falls within

Note: fields withinPhase, withinGroup and withinSubgroup represent the containment grouping 'parent' - and are mutually exclusive for context. If more than one of these fields is actually populated, then the order of preference for the parent element is: 1 - withinSubgroup, 2 - withinGroup, 3 - withinPhase.

Delimited data import/export format - Context stratigraphy records
Field name Description
siteCode Identifier of the site this stratigraphic record relates to
sourceID Identifier of the source context
stratRelationship Stratigraphic relationship (from source to target). Possible values: "above" (default), "below", "equal".
targetID Identifier of the target context

The stratigraphy record is a directional relationship between contexts (from source to target).

Delimited data import/export format - Context stratigraphy records (alternative '5 column' format)
Field name Description
siteCode Identifier of the site this stratigraphic record relates to
contextNo Identifier of the source context
contextType Type of context e.g. "layer", "cut", "fill" etc.
stratRelationship Stratigraphic relationship (from source to target). Possible values: "above" (default), "below", "equal".
relatedContextNo Identifier of the target context

An alternative stratigraphy record import format used in earlier work on the PHASER application, this option remains present for backward compatibility with earlier datasets already converted to this format for testing purposes.

Delimited data import/export format - Dating records
Field name Description
siteCode Identifier of the site this Dating record relates to
datingID Unique (within the site) identifier for this Dating record
datingType Type of Dating record
minYear Minimum year of the date range for this Dating record
maxYear Maxmum year of the date range for this Dating record
description Textual description of the Dating record
withinPeriod Identifier of the period associated with this Dating record
withinCOntext Identifier of the context this Dating record falls within
Delimited data import/export format - Period records
Field name Description
periodID Unique (within the site) identifier for this period record
minYear Minimum year of the date range for this period record
maxYear Maxmum year of the date range for this period record
description Textual description of the period record
uri URI associated with this period record

View - Zoom in

Increase the scale of the diagram to 'zoom in' on a particular area. Note the same functionality can alternatively be achieved using the slider to the left side of the diagram (slide up to zoom in, or click the '+' button at the top of the slider), or by using the mouse wheel if available. Zooming in will occur even if there are no elements currently present in the diagram. There are maximum/minimum zoom extents set to prevent zooming the diagram to an unusable state, although these extents may be exceeded by the zoom to fit function on a larger diagram.

[Back to top]

View - Zoom out

Decrease the scale of the diagram to 'zoom out' to view a wider area. Note the same functionality can alternatively be achieved using the slider to the left side of the diagram (slide down to zoom out, or click the '-' button at the bottom of the slider), or by using the mouse wheel if available. Zooming out will occur even if there are no elements currently present in the diagram. There are maximum/minimum zoom extents set to prevent zooming the diagram to an unusable state, although these extents may be exceeded by the 'zoom to fit' function on a larger diagram.

[Back to top]

View - Zoom to fit

Increase or decrease the scale of the diagram to make all contents fit within the current window.

[Back to top]

View - Refresh diagram

Periodically during data editing the diagram may go out of step with the data displayed in the tabs on the right hand side. This option will refresh the data shown in the diagram. The difference between this option and redo layout is that refreshing the diagram only syncs the data, it does not perform any repositioning of elements.

[Back to top]

View - Lock/Unlock diagram

By default the diagram is 'locked', preventing inadvertent repositioning of individual elements. Whilst locked the diagram can still be panned and zoomed as required. To change the position of an individual element on the diagram unlock the diagram, reposition the element, then lock the diagram again. Also the 'redo layout' functionality cannot be performed whilst the diagram is locked. This menu option will be either "Lock diagram" or "Unlock diagram" - depending on the current lock status.

[Back to top]

View - Redo layout

This will recalculate ALL element positions according to the DAGRE directed graph layout algorithm. The 'redo layout' functionality cannot be performed whilst the diagram is locked, and the menu option will be greyed out (disabled) accordingly. This option will automatically perform a refresh prior to calculating element positions, to ensure the diagram has all the latest data.

Matrix diagram

The diagram visually represents a matrix of contexts and the stratigraphic relationships between them, plus containment relationships (grouping and phasing). There are pan/zoom controls to the left hand side, a button to lock the diagram and a button to refresh the diagram data. There is a small expandable legend at the top right of the diagram helping to distinguish the various elements shown on the diagram. The background of the diagram is a light grey grid which helps with manual layout of elements, and the elements when dragged should 'snap to' (align with) this grid. (Note you can only drag elements on the diagram when the diagram is unlocked).

Tabs

The tabs hold data about the elements of the stratigraphic graph - phases, groups, subgroups, contexts, datings, periods and stratigraphic relationships - allowing reviewing and editing, each of these tabs also displays in the title a total count of the elements of this type. There are also other tabs for validation, analysis of temporal relationships, metadata about the current dataset, search and group matrix data. The tabs react to selection of elements (either in the diagram, or the table, or from other tabs) to display the details of the currently selected element, allowing editing.

Phases

The table lists all phase records in the current dataset, table columns can be sorted and the data can be filtered. Phases can be added, edited or deleted. A date range can be manually set for a phase, and compared to a date range derived from the elements within the phase.

[Back to top]

The table lists all group records in the current dataset, table columns can be sorted and the data can be filtered. Groups can be added, edited or deleted.

[Back to top]

[Back to top]

Validation

Various pre-defined validation checks can be performed on phases, groups, subgroups, contexts and periods to improve consistency. The results are colour coded to indicate where there may be potential problems within the dataset.

[Back to top]

Temporal

Tables displaying temporal vs stratigraphic relationships between elements contined within the selected phase. This helps to visualise whether the stratigraphic relationships supported by the dating evidence (and vice versa). Colour coding of table rows is used to indicate where there is a divergence between what the stratigraphy says and what the dating evidence says.

[Back to top]

Metadata

General descriptive metadata about (and stored with) the current dataset - fields contained within this section are optional. It is useful to record some supplementary metadata to keep track of the origins of the dataset, any revisions you may have made, and licensing and contact details if you are sharing the data with others.

[Back to top]

Search (experimental)

Search facilities to identify groups of elements based on hierarchical containment relationships (e.g. find all contexts and/or subgroups within a particular phase)

[Back to top]

Group Matrix (experimental)

The group matrix represents stratigraphic relationships between groups, which are derived from the stratigraphic relationships between contained contexts.
